Git & GitHub

Git is a version control system that allows you to track changes in your code and collaborate with others.
GitHub is a platform that hosts Git repositories and provides tools for collaboration.

Key Concepts

  • Repository: A collection of files and their history.
  • Commit: A snapshot of your repository at a point in time.
  • Branch: A parallel version of your repository.
  • Merge: Combining changes from different branches.
  • Clone: Combining changes from different branches.
  • Fork: Combining changes from different branches.

Git Commands: Overview and Usage

  • git init ➡️ Initialize a new Git repository.
  • git status ➡️ Show the working tree status.
  • git add 'file name' ➡️ Stage changes for commit.
  • git add . ➡️ Stage changes for commit all files.
  • git commit -m "Added/Updated/Deleted" ➡️ Commit staged changes.
  • git log ➡️ Show commit logs.
  • git push -u origin 'branch name' ➡️ Push changes to a remote repository.
  • git push origin 'branch name'
  • git branch ➡️ List, create, or delete branches.
  • git checkout ➡️ Switch branches or restore working tree files.
  • git merge ➡️ Merge changes from one branch into another.
  • git remote ➡️ Manage remote repositories.
  • git fetch + merge ➡️ Download objects and refs from another repository.
  • git pull ➡️ Fetch from a remote repository and merge.
  • git clone 'repository URL' ➡️ Clone a repository into a new directory.
  • git rm 'file name' ➡️ Remove files from the working tree and index.
  • git remote remove origin ➡️ Remove repository.
